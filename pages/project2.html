프로젝트 개요

출시를 목표로 한 프로젝트가 아니라, UI/UX 구조 개선, 개발 생산성 점검, 그리고 개발 과정에 대한 회고 기록에 초점을 맞춘 프로젝트입니다.

담당 역할

UI/UX 재정비 및 개선

  • 소울디바이스 개발 당시 작성한 UI/UX 코드 리팩토링
  • 레거시 구조 분석 및 개선
  • UI 애니메이션 연출 재조정 및 최적화
  • 폰트 리소스 용량 최적화

개발 진행 정리 및 회고

  • 주간 단위 작업 목표 설정
  • 개발 진행 과정 정리 및 공유
  • 주 5회 개발 일지 작성 (판단 과정 기록)

핵심 작업 내용

개발 일지 작성

주 5회, 모든 개발 과정을 상세히 기록하며 구현 과정뿐 아니라 설계 판단 근거, 발생한 문제, 개선 방향까지 문서화. 감으로 하던 개발을 언어화하고 정리하는 경험.

UI/UX 구조 개선

높은 의존성을 가진 레거시 UI 구조를 기능 단위로 분리하고, DOTween 애니메이션의 타이밍과 easing을 일관되게 재정의하여 완성도 향상.

문제 해결 경험

레거시 UI 구조로 인한 유지보수 어려움

문제: 높은 의존성으로 수정 범위 파악 어려움

해결: UI 요소를 기능 단위로 분리, 불필요한 의존성 제거

결과: UI 수정 범위 축소, 구조 이해도 향상

UI 애니메이션 연출 일관성 부족

문제: 타이밍과 easing이 일관되지 않아 완성도 저하

해결: DOTween 애니메이션 전반 점검, easing 함수 재정의

결과: UI 인터랙션 자연스러움 향상

배운 점

  • UI/UX 설계에서 구조와 유지보수성을 함께 고려하는 시야 확보
  • 개발 판단을 문서화하며 사고 과정을 정리하는 습관 형성
  • 감각에 의존하던 UI/UX 작업을 논리적으로 설명할 수 있게 됨
  • 협업 프로젝트에서도 개발 일지와 기록이 중요한 자산이 될 수 있음
