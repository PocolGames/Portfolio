UI/UX 시스템 구현

Unity UGUI 기반의 UI 시스템을 구현하였으며, 사용자 경험을 최우선으로 고려한 직관적인 인터페이스를 설계했습니다. 플레이어의 입력 동선을 최소화하고, 화면 전환 시 자연스러운 흐름을 유지하는 것을 핵심 설계 원칙으로 삼았습니다. 모든 UI 요소는 재사용 가능한 컴포넌트 구조로 설계되어, 유지보수성과 확장성을 확보했습니다.

// UI 매니저 기본 구조
public class UIManager : MonoBehaviour
{
    private Dictionary<string, UIPanel> panelDictionary;
    
    public void ShowPanel(string panelName)
    {
        if (panelDictionary.TryGetValue(panelName, out UIPanel panel))
        {
            panel.Show();
        }
    }
}
UI 패널을 Dictionary로 관리하여 빠른 접근과 효율적인 메모리 사용을 구현했습니다.

다국어 시스템

Unity Localization 패키지를 활용하여 7개 국어를 지원하는 다국어 시스템을 구축했습니다. 각 언어별 문화적 특성을 고려한 번역과 UI 레이아웃 조정을 통해 글로벌 사용자 경험을 최적화했습니다. AI 번역과 사전 자료를 병행하여 번역 품질을 확보하고, 특히 일본어의 경우 캐릭터 성격에 맞는 말투 표현을 적용했습니다.

다국어 지원 예시
언어 선택 UI
텍스트 레이아웃 조정
// 다국어 텍스트 적용 예시
using UnityEngine.Localization.Components;

public class LocalizedTextManager : MonoBehaviour
{
    [SerializeField] private LocalizeStringEvent localizeString;
    
    public void UpdateText(string key)
    {
        localizeString.StringReference.SetReference("UITexts", key);
    }
}
LocalizeStringEvent를 활용하여 런타임에 동적으로 텍스트를 변경할 수 있도록 구현했습니다.

DOTween 애니메이션

DOTween 라이브러리를 활용하여 부드럽고 직관적인 UI 애니메이션을 구현했습니다. 버튼 클릭, 패널 전환, 아이템 획득 등 다양한 상황에서 적절한 easing 함수를 적용하여 사용자 피드백을 강화했습니다. 애니메이션 시퀀스를 활용하여 복잡한 UI 연출도 간결한 코드로 구현할 수 있었습니다.

// DOTween을 활용한 UI 애니메이션
using DG.Tweening;

public void ShowPanelWithAnimation(RectTransform panel)
{
    panel.localScale = Vector3.zero;
    panel.DOScale(Vector3.one, 0.3f)
        .SetEase(Ease.OutBack);
    
    CanvasGroup canvasGroup = panel.GetComponent<CanvasGroup>();
    canvasGroup.alpha = 0;
    canvasGroup.DOFade(1f, 0.2f);
}
Scale과 Fade 애니메이션을 조합하여 자연스러운 패널 등장 효과를 구현했습니다.

// 로그라이트 스테이지 관리 시스템
public class StageManager : MonoBehaviour
{
    [SerializeField] private List<StageData> stages;
    private int currentStageIndex = 0;
    
    public void LoadNextStage()
    {
        currentStageIndex++;
        if (currentStageIndex < stages.Count)
        {
            LoadStage(stages[currentStageIndex]);
        }
    }
}
스테이지 기반 구조로 변경하여 개발 복잡도를 낮추고 완성도를 높였습니다.
