체스도쿠 (Chessdoku)

멀티플레이 시스템

Unity NetCode for GameObjects를 활용하여 P2P 방식의 멀티플레이 시스템을 구현했습니다. 턴제 게임 특성상 낮은 Tick Rate로도 원활한 플레이가 가능하도록 설계하여 서버 비용을 최소화했습니다. 클라이언트-서버 구조에서 핵심 게임 로직은 서버에서 처리하고, 클라이언트는 시각적 표현만 담당하도록 역할을 분리했습니다.

네트워크 구조
동기화 시스템
테스트 플레이
// NetCode 기반 네트워크 매니저
using Unity.Netcode;

public class GameNetworkManager : NetworkBehaviour
{
    [ServerRpc]
    public void MovePieceServerRpc(Vector2Int from, Vector2Int to)
    {
        // 서버에서 이동 검증
        if (IsValidMove(from, to))
        {
            ExecuteMove(from, to);
            UpdateBoardClientRpc(from, to);
        }
    }
}
서버에서 게임 로직을 검증하여 클라이언트 해킹을 방지했습니다.

게임 시스템 설계

체스와 스도쿠를 결합한 독창적인 게임 메커니즘을 설계했습니다. 시야 시스템, 안개 시스템, 코스트 및 레벨업 시스템 등 다양한 전략적 요소를 추가하여 게임의 깊이를 더했습니다. 스도쿠 빙고 완성 시 보너스 점수를 부여하는 시스템으로 목표 지향적인 플레이를 유도했습니다.

// 시야 시스템 구현
public class VisionSystem : MonoBehaviour
{
    public HashSet<Vector2Int> GetVisibleTiles(Piece piece)
    {
        var visibleTiles = new HashSet<Vector2Int>();
        var moveRange = piece.GetMoveRange();
        
        foreach (var tile in moveRange)
        {
            visibleTiles.Add(tile);
        }
        return visibleTiles;
    }
}
기물의 이동 범위를 시야로 활용하여 전략적 깊이를 추가했습니다.
