# UI/UX 재정비 및 개발 회고 프로젝트

> **기존 프로젝트의 UI/UX 구조를 재정비하고,  
> 개발 과정 전반을 개발 일지로 기록하며 작업 방식과 설계를 회고한 프로젝트**

## 1. 프로젝트 개요

본 프로젝트는 _소울디바이스_ 개발 이후,  
기존에 작성한 UI/UX 코드와 개발 방식을 되돌아보고 **재정비하기 위한 목적**으로 진행한 프로젝트입니다.

출시를 목표로 한 프로젝트가 아니라,  
**UI/UX 구조 개선, 개발 생산성 점검, 그리고 개발 과정에 대한 회고 기록**에 초점을 맞추었습니다.

- **플랫폼**: PC
- **장르**: 2D 로그라이크 프로토타입
- 개발 형태: 팀 프로젝트 (4명명)

## 2. 담당 역할 및 프로젝트 성격

### 주요 역할: UI/UX 재정비 및 개선

소울디바이스 개발 당시 작성한 UI/UX 코드를 가져와,  
**레거시 구조를 분석하고 개선하는 것을 핵심 목표**로 삼았습니다.

- 기존 UI/UX 코드 리팩토링 및 구조 개선
- UI 애니메이션 연출 재조정 및 최적화
- 폰트 리소스 용량 최적화 및 에셋 정리
- 재사용 가능성을 고려한 UI 구조 정리

### 보조 역할: 개발 진행 정리 및 회고 (PM적 역할)

출시 일정이 없는 프로젝트였기 때문에,  
**일정 관리보다는 ‘개발 방식 정리와 기록’에 집중**했습니다.

- 주간 단위 작업 목표 설정
- 개발 진행 과정 정리 및 공유
- 개발 중 판단과 문제 해결 과정을 개발 일지로 기록

## 3. 개발 일지 작성 및 개발 방식

본 프로젝트의 핵심은 **개발 일지 작성**이었습니다.

- 주 5회, 모든 개발 과정을 상세히 기록
- 구현 과정뿐 아니라
    - 왜 이 방식으로 설계했는지
    - 어떤 문제가 있었는지
    - 다음에는 어떻게 개선할 수 있을지를 함께 정리
- UI/UX 구조, 애니메이션 연출, 리팩토링 과정에 대한 판단을 문서로 정리

이를 통해 **감으로 하던 개발을 언어화하고 정리하는 경험**을 할 수 있었습니다.

## 4. 주요 개선 및 문제 해결 경험

### ① 레거시 UI 구조로 인한 유지보수 문제

**문제**  
초기 프로젝트에서 작성한 UI 구조는 의존성이 높아,  
작은 수정에도 여러 부분을 함께 수정해야 했습니다.

**개선**
- UI 구조를 다시 분석하여 불필요한 의존성 제거
- UI 요소를 기능 단위로 분리하여 재정비

**결과**
- UI 수정 범위 축소
- 구조 이해도 및 유지보수성 향상

### ② UI 애니메이션 연출의 일관성 부족

**문제**  
UI 애니메이션의 타이밍과 easing이 일관되지 않아 UX 완성도가 떨어졌습니다.

**개선**
- DOTween 애니메이션 전반 점검
- UI 성격에 맞는 easing 함수 재정의

**결과**

- UI 인터랙션의 자연스러움 향상
- UX 완성도 개선

## 5. 결과 및 배운 점

이 프로젝트를 통해 **결과물보다 과정이 중요한 개발 경험**을 할 수 있었습니다.

- UI/UX 설계에서 구조와 유지보수성을 함께 고려하는 시야 확보
- 개발 판단을 문서화하며 사고 과정을 정리하는 습관 형성
- 감각에 의존하던 UI/UX 작업을 논리적으로 설명할 수 있게 됨
- 협업 프로젝트에서도 개발 일지와 기록이 중요한 자산이 될 수 있음을 인식
