# 소울디바이스 (Soul Device)

> **UI/UX 개발을 중심으로 프로젝트에 참여했으며, 개발 후반부에는 일정 관리와 협업 프로세스 개선을 통해 Steam 출시까지 완료한 프로젝트**

## 1. 프로젝트 개요

**소울디바이스**는 2023년 12월 Steam에 출시한 **2D 횡스크롤 로그라이트 게임**입니다.  
플레이어는 전투를 통해 몬스터를 처치하고 아이템을 수집하며, 반복 플레이를 통해 보스에 도전하는 구조의 게임입니다.

- **플랫폼**: PC (Steam)
- **장르**: 2D 횡스크롤 로그라이트
- **개발 형태**: 팀 프로젝트 (7명)

## 2. 담당 역할 및 사용 기술

### 주요 역할: UI/UX 개발

기획자로부터 전달받은 기획 의도를 바탕으로, 디자이너와 협업하여 **UX 흐름을 기준으로 한 UI 구조 설계 및 Unity 기반 구현**을 담당했습니다.  
플레이 흐름을 방해하지 않도록 **UI 전환 단계와 입력 동선을 최소화하는 것**을 UX 설계의 기준으로 삼았습니다.

- Unity **UGUI** 기반 UI 시스템 구현
- **DOTween**을 활용한 UI 애니메이션 제작
- 다양한 해상도 및 화면 비율 대응 UI 레이아웃 구성
- Unity **Localization 패키지**를 활용한 **7개 국어 다국어 지원 시스템 구현**

### 보조 역할: PM (개발 후반부)

개발 후반부에 프로젝트 일정 지연과 협업 문제가 발생하면서,  
UI/UX 개발 외에도 프로젝트 완성을 위한 **협업 프로세스 개선과 일정 관리**를 병행했습니다.

- Git / GitHub 기반 협업 환경 도입 및 정착
- 작업 우선순위 재정립 및 개발 일정 조정
- 디자이너와의 일간 회의 진행 및 원격 협업 환경 구축
- 제한된 개발 기간 내 완성을 위한 개발 방향 조정 지원
  
## 3. 핵심 구현 내용

### UI/UX 시스템 구현

- UX 흐름을 기준으로 한 UI 구조 설계 및 화면 전환 로직 구현
- DOTween을 활용한 버튼 및 패널 전환 애니메이션 구현으로 직관적인 피드백 제공
- 다양한 해상도 환경에서도 UI 가독성을 유지할 수 있도록 레이아웃 구성

### 다국어(Localization) 시스템

- Unity Localization 패키지를 활용한 다국어 구조 설계
- AI 번역 및 사전 자료를 활용하여 7개 국어 번역 데이터 구축
- 언어별 특성을 고려한 번역 표현 적용
  (예: 일본어 히라가나 사용을 통한 어린아이 말투 표현 등)

## 4. 문제 해결 경험

### ① 협업툴 부재로 인한 개발 비효율 문제

**문제**  
개발 초기에는 협업툴이 없어 파일 최신화와 작업 공유가 원활하지 않았고, 이로 인해 작업 충돌과 개발 지연이 발생했습니다.

**해결**  
협업툴의 필요성을 인지하고 Git / GitHub 도입을 제안했으며,  
자료와 설명을 통해 팀원들을 설득하여 협업툴을 도입했습니다.

**결과**
- 작업 이력 관리 가능
- 파일 충돌 감소
- 팀원 간 작업 공유 및 커뮤니케이션 효율 향상

### ② 개발 일정 대비 과도한 개발 볼륨 문제

**문제**  
데드라인이 임박한 상황에서도 개발 진척이 느렸고, 기존 메트로바니아 장르 설계로는 프로젝트 완성이 어려운 상황이었습니다.

**해결**  
PM 역할을 맡아 개발 방향을 재정립하고,  
**메트로바니아에서 로그라이트 장르로 방향을 전환**하여 개발 볼륨을 조정했습니다.

**결과**
- 핵심 재미 요소에 집중한 구조로 변경
- 제한된 기간 내 프로젝트 완성 및 Steam 출시 달성

### ③ 개발 변경으로 인한 팀 사기 저하 문제

**문제**  
개발 방향 변경으로 인해 기존 작업물이 다수 폐기되며, 디자이너들의 개발 의욕이 저하되었습니다.

**해결**
- 디자이너들과 실시간으로 소통하며 작업 방향을 명확히 공유
- 기획자의 추상적인 의도를 구체화하여 디자이너에게 전달
- 피드백 반영 속도를 높여 작업 몰입도를 유지
**결과**
- 팀 내 소통 개선
- 개발 속도 및 작업 완성도 향상

## 5. 결과 및 배운 점

대학생 초반, 실력과 정보가 충분하지 않은 상태에서 시작한 비교적 큰 규모의 프로젝트를 통해  
**협업과 개발 프로세스의 중요성**을 깊이 이해하게 되었습니다.

- UI/UX 개발에서 사용자 흐름과 입력 동선을 고려한 설계의 중요성
- DOTween 및 다양한 패키지 활용을 통한 UI 완성도 향상 경험
- 다국어 지원 과정에서 언어별 표현 방식과 문화적 차이에 대한 이해
- 협업툴과 커뮤니케이션 환경이 프로젝트 완성도에 미치는 영향에 대한 경험

이 프로젝트를 통해 **UI/UX 개발을 중심으로 하되, 프로젝트 전체를 고려하며 문제를 해결하는 개발자**로 성장할 수 있었습니다.
