# 슬롯 꼬치 (Slot Ggochi)

> **AI를 전면 도입하여 기획부터 개발, 출시까지  
> 단 6일 만에 완성한 하이퍼캐주얼 게임 프로젝트**

## 1. 프로젝트 개요

**슬롯 꼬치**는 화면 터치로 슬롯을 멈추며 음식을 매칭하는 하이퍼캐주얼 슬롯 머신 게임입니다.

이 프로젝트의 핵심은 **AI를 활용한 초고속 개발**입니다.  
기획, 디자인, 개발, 광고 연동까지 모든 과정을 **AI와 협업하여 6일 만에 완성**했으며,  
이를 통해 **AI 시대의 게임 개발 생산성**을 실험하고 검증했습니다.

- **플랫폼**: Android (Google Play Store)
- **장르**: 하이퍼캐주얼 슬롯 게임
- **개발 기간**: 2025년 1월 5일 ~ 1월 10일 (6일)
- **개발 형태**: 개인 프로젝트 (AI 협업)
- **현재 상태**: Google Play Store 비공개 테스트 중

## 2. 프로젝트 목적 및 배경

**AI 도입 결정**
- AI를 활용하면 단순 버그를 사전에 방지하고 개발 시간을 단축할 수 있다고 판단
- 코드 리뷰, 버그 예측, 자동 수정 등 AI의 강점을 활용
- 전체 개발 프로세스에 AI를 통합하여 생산성 향상 실험

### AI 협업 방식

#### ① 토큰 효율 관리 전략

AI 활용의 가장 큰 과제는 **토큰 소비 최적화**였습니다.

**문제점**
- Unity 프로젝트 전체를 AI에 연결하면 파일 읽기만으로 막대한 토큰 소비
- 불필요한 파일(에셋, 메타파일 등)까지 읽으면 효율성 급격히 감소

**해결 방법**
- **filesystem 접근 범위를 스크립트 폴더로 제한**
- AI가 필요한 코드 파일만 선택적으로 읽도록 구성
- 결과: 토큰 소비를 대폭 절약하며 효율적인 협업 가능

```
[제한된 접근 구조]
Unity Project/
├── Assets/
│   └── Scripts/  ← AI가 접근 가능한 유일한 폴더
│       ├── GamePlayManager.cs
│       ├── SlotSpinController.cs
│       ├── AdMobManager.cs
│       └── ...
├── Packages/
├── ProjectSettings/
└── ... (AI 접근 불가)
```

**효과**
- 토큰 소비 90% 이상 절감
- AI 응답 속도 향상
- 필요한 코드만 집중적으로 관리

#### ② 개발 프로세스별 AI 활용

**기획 단계**
- 하이퍼캐주얼 장르 분석 및 핵심 메커니즘 설계
- 게임 룰, 난이도 곡선, 수익화 모델 기획

**개발 단계**

- 버그 예측 및 사전 방지

**광고 연동 단계**

## 4. 게임 시스템 설계

### 핵심 게임 메커니즘

**슬롯 머신 시스템**
- 3개의 슬롯 컬럼이 동시에 회전
- 화면 터치로 순차적으로 슬롯 정지
- 3개가 모두 매칭되면 성공, 하나라도 다르면 실패

**난이도 시스템**
- 점수가 오를수록 슬롯 회전 속도 증가
- 공식: `spinDuration = Clamp(0.5f - score * 0.02f, 0.1f, 0.5f)`
- 20점 도달 시 최고 난이도

**체력 시스템**
- 시작 시 하트 3개 제공
- 매칭 실패 시 하트 1개 감소
- 하트 0개 시 게임 오버

**보상형 광고 시스템**
- 게임 오버 시 "Again" 버튼 (보상형 광고)
- 광고 시청 후 하트 1개 회복 및 게임 재개
- 점수는 유지되어 연속 플레이 가능

### 기술 구현

**관리자 패턴 기반 구조**
```
GamePlayManager (게임 전체 흐름 관리)
├── SlotSpinController (3개 슬롯 통합 제어)
│   └── SlotColumn × 3 (개별 슬롯 회전/정지)
├── TouchInputHandler (터치 입력 처리)
├── FoodAnimationController (매칭 성공 애니메이션)
├── AudioManager (사운드 관리)
└── AdMobManager (광고 관리)
```

**이벤트 기반 시스템**
- 슬롯 정지, 스핀 완료 등을 이벤트로 처리
- 느슨한 결합으로 유지보수성 향상
- AI가 제안한 구조를 그대로 적용

```csharp
// 이벤트 기반 통신 예시
public event Action OnSpinStarted;
public event Action<int> OnSlotStopped;
public event Action<bool, int[]> OnSpinCompleted;
```

## 5. 수익화 시스템 구축

### 플랫폼별 빌드 구성 관리

**#define을 활용한 멀티 플랫폼 전략**

```csharp
#if PLAYSTORE_BUILD
    // Google Play Store 전용 코드
    // Google AdMob 광고 사용
#elif INAPTOS_BUILD
    // 인앱토스 전용 코드
    // 인앱토스 IAA 광고 사용 (준비 중)
#else
    // 개발 모드
    // 광고 없이 바로 실행
#endif
```

**장점**
- 하나의 코드베이스로 여러 플랫폼 관리
- 플랫폼별 광고 SDK 분리
- 빌드 설정만으로 플랫폼 전환 가능

### Google AdMob 광고 시스템

**광고 종류 및 활용**
| 광고 종류 | 표시 위치 | 목적 |
|---------|---------|------|
| 배너 광고 | 메인 메뉴 하단 | 지속적 노출 |
| 전면 광고 | 게임 재시작 시 | 수익 극대화 |
| 보상형 광고 | 게임 오버 (Again 버튼) | 체력 회복 |

**AdMobManager 구조**
```csharp
public class AdMobManager : MonoBehaviour
{
    // 싱글톤 패턴
    public static AdMobManager Instance { get; private set; }
    
    // 광고 단위 ID (실제 운영용)
    [SerializeField] private string bannerAdUnitId;
    [SerializeField] private string interstitialAdUnitId;
    [SerializeField] private string rewardedAdUnitId;
    
    // 광고 로드 및 표시
    public void ShowRewardedAd(Action onAdCompleted);
    public void ShowInterstitialAd(Action onAdCompleted);
    public void ShowBannerAd();
}
```

**광고 흐름**
1. 게임 시작 시 AdMob 초기화
2. 전면 광고, 보상형 광고 미리 로드
3. 적절한 타이밍에 광고 표시
4. 광고 완료 후 콜백 실행
5. 다음 광고 미리 로드 (자동)

### 수익화 전략

**하이퍼캐주얼 게임 수익 모델**
- 무료 다운로드 + 광고 수익
- 짧은 플레이 타임, 높은 반복성 → 광고 노출 빈도 증가
- 보상형 광고로 유저 이탈 방지 및 수익 증대

**예상 수익 구조**
- 배너 광고: 낮은 단가, 지속적 노출
- 전면 광고: 높은 단가, 게임 재시작 시마다 노출
- 보상형 광고: 가장 높은 단가, 유저의 자발적 시청

## 8. 문제 해결 경험

### ① 토큰 소비 최적화

**문제**
- Unity 프로젝트 전체를 AI에 연결하면 토큰 소비 폭발
- 초기 파일 읽기만으로도 수천 토큰 소비

**해결**
- filesystem 접근 범위를 Scripts 폴더로 제한
- 필요한 파일만 선택적으로 AI에게 제공
- 결과: 토큰 소비 90% 이상 절감

**교훈**
- AI 활용 시 **컨텍스트 관리**가 핵심
- 필요한 정보만 제공하는 것이 효율적
- 프로젝트 구조를 AI 친화적으로 설계하는 것이 중요

### ② 플랫폼별 빌드 관리

**문제**
- Google Play Store와 인앱토스를 모두 지원해야 함
- 각 플랫폼별로 다른 광고 SDK 사용

**해결**
- #define을 활용한 조건부 컴파일
- 하나의 코드베이스로 여러 플랫폼 관리
- BuildConfigurationManager로 빌드 설정 자동화

**교훈**
- 멀티 플랫폼 전략은 설계 단계부터 고려해야 함
- 조건부 컴파일을 활용한 코드 분리 효과적

### ③ AI와의 효율적인 협업 방법

**초기 시행착오**
- 너무 추상적인 요청 → 원하는 결과 나오지 않음
- 너무 구체적인 요청 → 토큰 낭비

**최적의 협업 방식**
1. 큰 틀의 구조를 먼저 AI와 논의
2. 구체적인 구현은 단계적으로 요청
3. 생성된 코드를 검토하고 피드백
4. 반복적 개선을 통한 최적화

**교훈**
- AI는 **협업 파트너**이지 자동화 도구가 아님
- 명확한 요구사항 정의가 핵심
- AI의 제안을 비판적으로 검토하는 능력 필요
