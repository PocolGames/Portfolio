# Firebase 서비스 결정 재고

## 내용

Firebase를 사용하려고 했으나, 로그인 시스템 부분에서 유니티 로그인 서비스를 이용하는 것이 더 합리적일 수도 있다고 판단하였음.

SNS 로그인을 기능같은 부분에서 유니티 로그인 서비스가 더 다양하게 지원하고 있음.

## 결론

유니티에서 지원하는 여러가지 기능인 Relay, Lobby, Matchmaker 등과 자동으로 연동하기 때문에 유니티 서비스를 더 이용하기로 결정.

정리해서 회의록용으로 깔끔하게 다듬으면 이렇게 쓸 수 있습니다.

# 로그인 구현 시스템 난이도

## 플랫폼 제한

* 스팀 로그인: 스팀 플랫폼 출시 시에만 가능
* 애플 로그인: 애플 디바이스에서만 사용 가능

## 현재 구현 가능 로그인 방식

* 기본 회원가입/로그인
* 구글 로그인
* 페이스북 로그인
* 게스트 로그인 (추가 필요)

## 참고 사항

* 많은 인디게임은 초기 단계에서 게스트 로그인과 기본 회원가입/로그인만 구현하고, 이후 필요에 따라 다른 SNS 로그인을 추가하는 경우가 많음 (구현 복잡도 때문)
* 대기업 게임은 초기부터 모든 SNS 로그인 기능을 구현하여 제공
