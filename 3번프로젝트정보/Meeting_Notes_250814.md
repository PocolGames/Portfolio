# Firebase 서비스 결정

## 내용

* 로그인 시스템, 랭킹 시스템을 구현하기 위해서 백엔드 서비스를 선택해야됨.

## **Firebase를 선택한 이유**

1. **서버 개발 및 유지보수 부담 감소**

   * Firebase는 **BaaS(Backend as a Service)** 플랫폼으로, 서버를 직접 구축하거나 유지보수할 필요 없이 인증, 데이터베이스, 파일 저장 등 핵심 기능을 바로 사용할 수 있습니다.
   * 소규모 개발팀이나 1인 개발 환경에서 시간과 인력을 절약할 수 있습니다.

2. **Unity 공식 지원 및 SDK 제공**

   * Firebase는 Unity용 SDK를 공식적으로 제공하여 Android, iOS, PC 등 멀티 플랫폼 개발 시 호환성이 좋습니다.
   * 예제 코드와 문서가 풍부해 개발 속도를 높일 수 있습니다.

3. **실시간 데이터 처리 기능**

   * **Realtime Database**와 **Cloud Firestore**를 통해 데이터가 변경되면 모든 클라이언트에 실시간으로 동기화됩니다.
   * 멀티플레이, 채팅, 실시간 순위표 같은 기능 구현에 유리합니다.

4. **다양한 인증 방식 지원**

   * 이메일/비밀번호, 구글 계정, 페이스북, 애플 로그인 등 여러 로그인 방식을 손쉽게 구현할 수 있습니다.
   * 보안 및 토큰 관리도 Firebase에서 자동으로 처리해줍니다.

5. **무료 플랜(Spark)으로 시작 가능**

   * 초기 개발 단계에서 별도의 서버 비용 없이 무료로 테스트 및 운영이 가능합니다.
   * 필요에 따라 유료 플랜으로 확장할 수 있습니다.

6. **보안 규칙 기반 데이터 보호**

   * Firebase는 보안 규칙(Security Rules) 기능을 제공하여, 유저 권한에 따른 데이터 접근 제어를 쉽게 구현할 수 있습니다.
   * 별도의 서버 측 권한 로직을 작성하지 않아도 안전한 데이터 관리가 가능합니다.

7. **Google Cloud 인프라 기반 안정성**

   * 구글 클라우드 인프라를 기반으로 제공되기 때문에, 높은 안정성과 확장성을 보장합니다.
   * 전 세계 여러 리전에 서버를 두어 글로벌 서비스 운영이 가능합니다.
