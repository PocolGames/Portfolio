# 안개시스템 보안 분석

![ScreenShot](/docs/image/03_Image_Meeting_Notes_Documents/00_client_vision_system_comparison.png)

## 주제

안개 시스템 작동 방식

### 완료된 작업

- 시점 변경 구현 완료
- 클라이언트별 개인 시야 적용 완료

### 진행 중

- OrderLayer 수치 조정 - 기물, 안개, 기본 땅 등의 렌더링 순서 조정 필요

### 검토 필요  

- 안개 시스템 구현 방식 결정(기물과 숫자 표시 방법): 물리적으로 안개로 가리는 방식 vs 시스템상 On/Off 처리 방식

## 검토

### 문제점 분석

- 물리적 안개 가리기 방식의 보안 취약점 문제.
- 마인크래프트 X-ray 핵과 동일한 원리로 악용 가능.
- 클라이언트가 안개 텍스쳐를 투명하게 수정하여 숨겨진 기물 확인 가능.
- 클라이언트 측에서 랜더링만 차단하는 방식은 근복적으로 우회 가능.
- 또한, 기본적인 안개 텍스쳐가 타일을 모두 가리지 않을 수 있음. (ex - 불투명한 안개)

### 제안된 해결책 

- 서버 기반 이미지 처리
- 시스템 상 On/Off 방식

### 결론

- 시스템 상 On/Off 방식이 보안 측면에서 더 안전하다는 판단이 맞다고 생각하지만, 동기화 제한과 성능 최적화 부분에 대한 기술적 검토가 필요하다.
