# 인게임 UI 디자인

![Screenshot](/docs/image/01_Image_Design_Planning_Documents/09_ingame_status_ui_sketch.png)

## 종류 요약

* 플레이어 초상화 (좌측 상단)
* 플레이어 이름 (좌측 상단)
* 플레이어 레벨 (좌측 상단)
* 상대 플레이어 초상화 (우측 상단)
* 상대 플레이어 이름 (우측 상단)
* 상대 플레이어 레벨 (우측 상단)
* 현재 보유 코스트 (우측)
* 레벨업 버튼 (하단 6개 버튼 중 가장 왼쪽)
* 새로고침 버튼 (히딘 6개 버튼 중 가장 오른쪽)
* 기물 뽑는 버튼 4개 (하단 4개 버튼)
* 버튼 사용 코스트 (하단 6개 버튼의 우측 하단)
* 점수표 (상단 중앙)


## 추가 설명

### 플레이어 초상화

초상화는 필수 요소라고 생각하지 않지만, 있어야 밸런스가 잡힐 것 같은 느낌이 들어서 추가했음.

### 레벨

플레이어 본인의 레벨을 확인하는 것은 당연하지만 상대방의 레벨을 보는 것에 대해서 고민중이다. 

상대방의 레벨을 보여주게 되면, 상대방의 레벨이 오르는 것을 보고 그 턴에서 새로운 기물을 뽑지 않았다는 것을 예측할 수 있다.

이런 예측이 좋은 방향인지, 나쁜 방향인지 모르겠음.

### 현재 보유 코스트

코스트 UI의 위치를 어느 곳에 배치해야 이상적인지 모르겠음.
현재는 애매하게 오른쪽 중단도 아닌 약간 아래임.

### 레벨업 버튼, 새로고침 버튼

레벨업 버튼과 새로고침 버튼의 모양을 나머지 버튼(기물 뽑는 버튼 6개)와 다르게 디자인을 하는 것이 좋다고 생각함.

예를들면, 기물 뽑는 버튼이 동그라미, 레벨업 버튼과 새로고침 버튼은 네모.

### 기물 뽑는 버튼 4개

이 버튼 안에는 기물의 초상화가 들어갈 예정이다.

### 버튼 사용 코스트

버튼 사용 코스트 UI의 위치는 코스트를 사용하는 하단 버튼 6개와 인접해있는 곳 아무 곳이면 된다.
위치는 똑같아야 된다.

### 점수표

상단 중앙. 
몇 대 몇의 스코어를 보여준다.

누가 우세한지 알려주는 ui가 추가될 수도 있다.
알려주는 정보는 간접적이다. 약간 우세, 우세, 많이 우세. 이렇게 보여준다.
직접적인 수치로 보여주면 플레이어가 자세하게 유추할 수 있을 것 같기 때문이다.
그래서, 이런 정보를 보여줘야 되는데, 텍스트도 괜찮고, 텍스트가 없는 디자인으로 보여줘도 된다.
