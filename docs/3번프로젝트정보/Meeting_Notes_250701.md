# 회의록

## 주요 결정사항

**개발 환경**
- Unity 6000.2.0b7 사용 (최신 버전 선택)
- 완전한 2D 게임으로 개발 진행
- 멀티플레이어는 P2P 방식으로 우선 구현
- Unity Cloud 활성화 필요 (AI 기능 사용)

**개발 방향**
- 게임 시스템 복잡도가 낮아 처음부터 재개발 시에도 부담이 적음
- 프로토타입 개발 우선, 프로젝트 세팅은 후순위
- 문서 작성 및 회의록 작성에 며칠간 집중 예정

## 게임 설계

**2D vs 3D 선택 근거**
- 보드 게임의 직관적 플레이를 위해 2D가 더 적합
- 개발 시간 단축 고려
- 단, 기물 스킨 판매 등 수익화 측면에서는 3D가 유리할 수 있음 (포트나이트 사례)

**서버 동기화 구조**
- 총 32개 기물 관리
- 서버에서 모든 핵심 로직 처리
  - 이동, 충돌 확인, 전투 처리, 변수 변경
- 클라이언트는 시각적 표현만 담당
- 동기화 요소: 기물 위치/체력/공격력, 이동/파괴/생성 이벤트

## 구현 예정 기능

**기물 이동 시스템**
- 각 기물별 이동 패턴 구현 (폰, 나이트, 비숍, 룩, 킹, 퀸)
- 기본 이동 메커니즘 (직선, 대각선 이동)

## ❓ 논의된 이슈

**모바일 UI/UX 고려사항**
- **문제**: 모바일에서 기물이 작게 보일 가능성
- **해결방안**: 줌인/아웃 기능, 시야 시스템 활용
- **추가 아이디어**: 기물 이동/공격 시 확대 카메라 액션 (후순위 적용 검토)

**카메라 시점**
- 완전한 체스 스타일이 아닌 쿼터뷰 검토
- 기물 스킨의 매력적 표현을 위한 시점 선택 필요
